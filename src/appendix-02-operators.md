## পরিশিষ্ট B: অপারেটর এবং প্রতীক

এই পরিশিষ্টে Rust এর সিনট্যাক্সের একটি শব্দকোষ রয়েছে, যেখানে অপারেটর এবং অন্যান্য প্রতীকগুলি রয়েছে যা একা বা পাথ, জেনেরিক, trait বাউন্ড, ম্যাক্রো, অ্যাট্রিবিউট, কমেন্ট, টাপল এবং ব্র্যাকেটের প্রেক্ষাপটে প্রদর্শিত হয়।

### অপারেটর

সারণী B-1 এ Rust-এর অপারেটর, একটি উদাহরণ যেখানে অপারেটরটি কন্টেক্সটে প্রদর্শিত হবে, একটি সংক্ষিপ্ত ব্যাখ্যা এবং সেই অপারেটরটি ওভারলোড করা যায় কিনা তা উল্লেখ করা হয়েছে। যদি কোনো অপারেটর ওভারলোড করা যায়, তাহলে সেই অপারেটরটিকে ওভারলোড করার জন্য প্রাসঙ্গিক trait তালিকাভুক্ত করা হয়েছে।

### নন-অপারেটর প্রতীক

নিম্নলিখিত তালিকায় সেই সমস্ত প্রতীক রয়েছে যা অপারেটর হিসাবে কাজ করে না; অর্থাৎ, সেগুলি কোনও ফাংশন বা মেথড কলের মতো আচরণ করে না।

সারণী B-2 তে এমন প্রতীক দেখানো হয়েছে যা একা প্রদর্শিত হয় এবং বিভিন্ন স্থানে বৈধ।

<span class="caption">সারণী B-2: স্বতন্ত্র সিনট্যাক্স</span>

| প্রতীক                                        | ব্যাখ্যা                                                                |
| --------------------------------------------- | ---------------------------------------------------------------------- |
| `'ident`                                      | নামযুক্ত লাইফটাইম অথবা লুপ লেবেল                                           |
| `...u8`, `...i32`, `...f64`, `...usize`, ইত্যাদি। | নির্দিষ্ট প্রকারের সংখ্যাসূচক লিটারেল                                       |
| `"..."`                                       | স্ট্রিং লিটারেল                                                         |
| `r"..."`, `r#"..."#`, `r##"..."##`, ইত্যাদি।      | র' স্ট্রিং লিটারেল, যেখানে escape ক্যারেক্টার প্রক্রিয়া করা হয় না                    |
| `b"..."`                                      | বাইট স্ট্রিং লিটারেল; স্ট্রিং এর পরিবর্তে বাইটের একটি অ্যারে তৈরি করে  |
| `br"..."`, `br#"..."#`, `br##"..."##`, ইত্যাদি।   | র' বাইট স্ট্রিং লিটারেল, র' এবং বাইট স্ট্রিং লিটারেলের সংমিশ্রণ    |
| `'...'`                                       | ক্যারেক্টার লিটারেল                                                      |
| `b'...'`                                      | ASCII বাইট লিটারেল                                                     |
| <code>&vert;...&vert; expr</code>             | ক্লোজার                                                                |
| `!`                                           | ডাইভার্জিং ফাংশনগুলির জন্য সর্বদা খালি বটম টাইপ                         |
| `_`                                           | “উপেক্ষিত” প্যাটার্ন বাইন্ডিং; ইন্টিজার লিটারেলগুলিকে পাঠযোগ্য করতেও ব্যবহৃত হয় |

সারণী B-3 তে সেই প্রতীকগুলি দেখানো হয়েছে যা মডিউল হায়ারার্কির মাধ্যমে কোনও আইটেমের পথে প্রদর্শিত হয়।

<span class="caption">সারণী B-3: পাথ-সম্পর্কিত সিনট্যাক্স</span>

| প্রতীক                                  | ব্যাখ্যা                                                                                                                    |
| --------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| `ident::ident`                          | নেমস্পেস পাথ                                                                                                                 |
| `::path`                                | এক্সটার্ন প্রি্লুডের সাপেক্ষে পাথ, যেখানে অন্যান্য সমস্ত ক্রেট রুট করা হয় (যেমন, ক্রেটের নাম সহ একটি সুস্পষ্ট অ্যাবসলিউট পাথ) |
| `self::path`                            | বর্তমান মডিউলের সাপেক্ষে পাথ (যেমন, একটি সুস্পষ্ট রিলেটিভ পাথ)।                                                            |
| `super::path`                           | বর্তমান মডিউলের প্যারেন্টের সাপেক্ষে পাথ                                                                                   |
| `type::ident`, `<type as trait>::ident` | অ্যাসোসিয়েটেড কনস্ট্যান্ট, ফাংশন এবং টাইপ                                                                                     |
| `<type>::...`                           | এমন একটি টাইপের জন্য অ্যাসোসিয়েটেড আইটেম যা সরাসরি নামকরণ করা যায় না (যেমন, `<&T>::...`, `<[T]>::...`, ইত্যাদি)       |
| `trait::method(...)`                    | যে trait এটি সংজ্ঞায়িত করে তার নাম দিয়ে একটি মেথড কলকে দ্ব্যর্থতা মুক্ত করা                                                                |
| `type::method(...)`                     | যে টাইপের জন্য এটি সংজ্ঞায়িত করা হয়েছে তার নাম দিয়ে একটি মেথড কলকে দ্ব্যর্থতা মুক্ত করা                                                          |
| `<type as trait>::method(...)`          | trait এবং type এর নাম দিয়ে একটি মেথড কলকে দ্ব্যর্থতা মুক্ত করা                                                                  |

সারণী B-4 এ জেনেরিক টাইপ প্যারামিটার ব্যবহারের প্রেক্ষাপটে প্রদর্শিত প্রতীকগুলি দেখানো হয়েছে।

<span class="caption">সারণী B-4: জেনেরিক</span>

| প্রতীক                         | ব্যাখ্যা                                                                                                                             |
| ------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------- |
| `path<...>`                    | একটি টাইপে জেনেরিক টাইপের প্যারামিটার নির্দিষ্ট করে (যেমন, `Vec<u8>`)                                                                    |
| `path::<...>`, `method::<...>` | একটি এক্সপ্রেশনে জেনেরিক টাইপ, ফাংশন বা মেথডের প্যারামিটার নির্দিষ্ট করে; প্রায়শই টার্বোফিশ হিসাবে উল্লেখ করা হয় (যেমন, `"42".parse::<i32>()`) |
| `fn ident<...> ...`            | জেনেরিক ফাংশন সংজ্ঞায়িত করুন                                                                                                          |
| `struct ident<...> ...`        | জেনেরিক স্ট্রাকচার সংজ্ঞায়িত করুন                                                                                                         |
| `enum ident<...> ...`          | জেনেরিক তালিকা সংজ্ঞায়িত করুন                                                                                                             |
| `impl<...> ...`                | জেনেরিক ইমপ্লিমেন্টেশন সংজ্ঞায়িত করুন                                                                                                     |
| `for<...> type`                | উচ্চ-স্তরের লাইফটাইম বাউন্ড                                                                                                            |
| `type<ident=type>`             | একটি জেনেরিক টাইপ যেখানে এক বা একাধিক অ্যাসোসিয়েটেড টাইপের নির্দিষ্ট অ্যাসাইনমেন্ট রয়েছে (যেমন, `Iterator<Item=T>`)                    |

সারণী B-5 এ trait বাউন্ডের সাথে জেনেরিক টাইপ প্যারামিটারগুলিকে সীমাবদ্ধ করার প্রেক্ষাপটে প্রদর্শিত প্রতীকগুলি দেখানো হয়েছে।

<span class="caption">সারণী B-5: Trait বাউন্ড কনস্ট্রেইন্ট</span>

| প্রতীক                        | ব্যাখ্যা                                                                                                                               |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| `T: U`                        | জেনেরিক প্যারামিটার `T`, `U` implement করে এমন টাইপগুলির মধ্যে সীমাবদ্ধ                                                                    |
| `T: 'a`                       | জেনেরিক টাইপ `T` কে লাইফটাইম `'a` এর থেকে বেশি সময় ধরে থাকতে হবে (অর্থাৎ, টাইপটি `'a` এর থেকে ছোট লাইফটাইম যুক্ত কোনো রেফারেন্স ধারণ করতে পারবে না) |
| `T: 'static`                  | জেনেরিক টাইপ `T`-তে `'static` রেফারেন্স ছাড়া অন্য কোনো বরোড রেফারেন্স নেই                                                                 |
| `'b: 'a`                      | জেনেরিক লাইফটাইম `'b` কে লাইফটাইম `'a` এর থেকে বেশি সময় ধরে থাকতে হবে                                                                    |
| `T: ?Sized`                   | জেনেরিক টাইপ প্যারামিটারকে ডাইনামিকালি সাইজড টাইপ হওয়ার অনুমতি দিন                                                                      |
| `'a + trait`, `trait + trait` | কম্পাউন্ড টাইপ কনস্ট্রেইন্ট                                                                                                                 |

সারণী B-6 এ ম্যাক্রো কল করা বা সংজ্ঞায়িত করার এবং কোনও আইটেমের উপর অ্যাট্রিবিউট নির্দিষ্ট করার প্রেক্ষাপটে প্রদর্শিত প্রতীকগুলি দেখানো হয়েছে।

<span class="caption">সারণী B-6: ম্যাক্রো এবং অ্যাট্রিবিউট</span>

| প্রতীক                                      | ব্যাখ্যা        |
| ------------------------------------------- | ----------- |
| `#[meta]`                                   | বাইরের অ্যাট্রিবিউট    |
| `#![meta]`                                  | ভেতরের অ্যাট্রিবিউট    |
| `$ident`                                    | ম্যাক্রো প্রতিস্থাপন |
| `$ident:kind`                               | ম্যাক্রো ক্যাপচার      |
| `$(…)…`                                     | ম্যাক্রো পুনরাবৃত্তি   |
| `ident!(...)`, `ident!{...}`, `ident![...]` | ম্যাক্রো ইনভোকেশন   |

সারণী B-7 এ কমেন্ট তৈরি করে এমন প্রতীকগুলি দেখানো হয়েছে।

<span class="caption">সারণী B-7: কমেন্ট</span>

| প্রতীক     | ব্যাখ্যা             |
| ---------- | ----------------------- |
| `//`       | লাইন কমেন্ট            |
| `//!`      | ভেতরের লাইন ডক কমেন্ট  |
| `///`      | বাইরের লাইন ডক কমেন্ট  |
| `/*...*/`  | ব্লক কমেন্ট           |
| `/*!...*/` | ভেতরের ব্লক ডক কমেন্ট |
| `/**...*/` | বাইরের ব্লক ডক কমেন্ট |

সারণী B-8 এ টাপল ব্যবহারের প্রেক্ষাপটে প্রদর্শিত প্রতীকগুলি দেখানো হয়েছে।

<span class="caption">সারণী B-8: টাপল</span>

| প্রতীক                   | ব্যাখ্যা                                                                                 |
| ------------------------ | ------------------------------------------------------------------------------------------- |
| `()`                     | খালি টাপল (ইউনিট নামেও পরিচিত), লিটারেল এবং টাইপ উভয়ই                                               |
| `(expr)`                 | প্যারেনথেসাইজড এক্সপ্রেশন                                                                    |
| `(expr,)`                | সিঙ্গেল-এলিমেন্ট টাপল এক্সপ্রেশন                                                             |
| `(type,)`                | সিঙ্গেল-এলিমেন্ট টাপল টাইপ                                                                   |
| `(expr, ...)`            | টাপল এক্সপ্রেশন                                                                            |
| `(type, ...)`            | টাপল টাইপ                                                                                  |
| `expr(expr, ...)`        | ফাংশন কল এক্সপ্রেশন; টাপল `struct` এবং টাপল `enum` ভেরিয়েন্ট শুরু করতেও ব্যবহৃত হয় |
| `expr.0`, `expr.1`, ইত্যাদি। | টাপল ইনডেক্সিং                                                                              |

সারণী B-9 এ সেই প্রেক্ষাপটগুলি দেখানো হয়েছে যেখানে কার্লি ব্রেস ব্যবহার করা হয়।

<span class="caption">সারণী B-9: কার্লি ব্র্যাকেট</span>

| কন্টেক্সট      | ব্যাখ্যা      |
| ------------ | ---------------- |
| `{...}`      | ব্লক এক্সপ্রেশন |
| `Type {...}` | `struct` লিটারেল |

সারণী B-10 এ সেই প্রেক্ষাপটগুলি দেখানো হয়েছে যেখানে স্কোয়ার ব্র্যাকেট ব্যবহার করা হয়।

<span class="caption">সারণী B-10: স্কোয়ার ব্র্যাকেট</span>

| কন্টেক্সট                                            | ব্যাখ্যা                                                                                                                      |
| -------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `[...]`                                            | অ্যারে লিটারেল                                                                                                                  |
| `[expr; len]`                                      | `expr`-এর `len` সংখ্যক কপি ধারণকারী অ্যারে লিটারেল                                                                            |
| `[type; len]`                                      | `type`-এর `len` সংখ্যক উদাহরণ ধারণকারী অ্যারে টাইপ                                                                              |
| `expr[expr]`                                       | কালেকশন ইনডেক্সিং। ওভারলোডযোগ্য (`Index`, `IndexMut`)                                                                         |
| `expr[..]`, `expr[a..]`, `expr[..b]`, `expr[a..b]` | কালেকশন স্লাইসিং হওয়ার ভান করে কালেকশন ইনডেক্সিং, "ইনডেক্স" হিসাবে `Range`, `RangeFrom`, `RangeTo` বা `RangeFull` ব্যবহার করে |
